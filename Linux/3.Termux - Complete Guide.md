# Termux - Complete Guide (Beginner → Advanced, Cybersecurity Edition)

A practical Termux guide that takes you from zero to running a real pentesting/CTF toolkit on your phone — trimmed down to what's actually useful, with cybersecurity-relevant tools woven in throughout.

> [!warning] Legal & Ethical Notice Every scanning, brute-forcing, or exploitation tool in this guide is for **systems you own** or have **explicit written permission** to test (your own lab, a CTF box, TryHackMe/HackTheBox/OverTheWire targets). Running these against random/real-world systems without permission is illegal in most countries. Treat your phone like a pentest laptop, not a toy.

---

## What is Termux?

**Termux** is a terminal emulator + Linux environment for Android — no root required.

**Why it matters for a security student:**

- A pocket Linux box for `ssh`-ing into OverTheWire/TryHackMe/HTB boxes when you're away from your laptop
- Lets you practice recon, scripting, and CTF tooling anywhere
- Same package manager (`apt`/`pkg`) and shell habits you already use on your Arch box transfer directly

**Key Features:**

- No root required
- `apt`/`pkg` package manager with thousands of Linux packages
- Built-in SSH client and server
- Python, Node.js, and scripting support
- Access to the Android filesystem

## Part 1 — Beginner: Core Setup

### Step 1: Installation

1. Install via **F-Droid** (recommended — Play Store build is outdated/unmaintained)
2. Search "Termux", install, open it

```bash
# Termux sets itself up automatically on first launch
$
```

### Step 2: The Interface

- **Command prompt (`$`)** — where you type
- **Virtual keyboard row** — extra keys (Ctrl, Alt, arrows, Tab)
- **Volume Down + letter** — shortcut for Ctrl combos (see cheat sheet below)
- **Swipe from left edge** — drawer (sessions, "new session", keyboard toggle)
- **Long press** — copy/paste menu

### Step 3: Update & Upgrade

```bash
pkg update && pkg upgrade -y
```

- `pkg update` — refresh package lists
- `pkg upgrade` — install newer versions
- `-y` — auto-confirm prompts

**Run this weekly** and before installing anything new — outdated packages are a common source of broken installs.

### Step 4: Navigation Basics

```bash
pwd                 # current directory
ls / ls -la         # list files / list all incl. hidden
cd Documents         # move into a folder
cd ..                # up one level
cd ~  or  cd         # home directory
mkdir my_folder       # create directory
rmdir my_folder       # remove empty directory
```

**Key paths:**

- `~` = `/data/data/com.termux/files/home` — your home
- `~/storage/shared` — internal storage (after Step 5)
- `~/storage/downloads` — Downloads folder

### Step 5: Storage Access

```bash
termux-setup-storage
```

Grants Termux access to Android storage. After running it:

```bash
cd ~/storage/downloads   # Downloads
cd ~/storage/shared      # Internal storage
cd ~/storage/dcim        # Camera photos
```

### Step 6: Essential Packages

```bash
pkg install nano vim git wget curl python openssh nmap -y
```

|Package|Why|
|---|---|
|`nano`|beginner-friendly text editor|
|`vim`|powerful editor (steep learning curve, worth it long-term)|
|`git`|version control — for your notes/scripts/CTF write-ups|
|`wget` / `curl`|download files, fetch URLs, test endpoints|
|`python`|scripting — automation, exploit dev, parsing|
|`openssh`|SSH client/server — needed for OverTheWire, remote boxes|
|`nmap`|network/port scanning — the recon workhorse|

### Step 7: Creating & Editing Files

```bash
nano myfile.txt        # Ctrl+O save, Ctrl+X exit
vim myfile.txt          # i = insert, Esc then :wq = save+quit

echo "text" > file.txt   # overwrite
echo "more" >> file.txt  # append
cat file.txt              # view
```

### Step 8: File Operations

```bash
touch newfile.txt
cp file.txt backup.txt
cp -r folder/ folder_backup/
mv old.txt new.txt
rm file.txt
rm -r folder/
cat file.txt          # full content
less file.txt          # paginated view (good for long logs)
```

### Step 9: Package Management

```bash
pkg search keyword       # find a package
pkg install name          # install
pkg uninstall name         # remove
pkg list-installed          # what's installed
pkg show name                # package details
pkg autoclean && pkg clean    # free up space
```

**Worth exploring:** `htop` (process viewer), `tree` (directory tree), `neofetch` (system info).

## Part 2 — Intermediate: Dev & Network Foundations

### Step 10: SSH (you'll use this constantly)

**As a client — connecting out (e.g. OverTheWire, your VPS, HTB boxes):**

```bash
ssh username@server_address
ssh -p 2220 bandit0@bandit.labs.overthewire.org   # custom port example
scp file.txt user@server:~/destination/             # upload
scp user@server:~/file.txt ./                        # download
```

**As a server — letting something connect into Termux:**

```bash
sshd                       # start SSH daemon (default port 8022)
passwd                      # set a password first — required
ifconfig                     # find your IP to connect with
ssh -p 8022 user@your_ip      # from the other device
pkill sshd                     # stop the server
```

### Step 11: Python for Security Scripting

```bash
pkg install python
pip install requests beautifulsoup4
```

```python
#!/data/data/com.termux/files/usr/bin/python
import requests
r = requests.get('https://api.github.com')
print(r.status_code)
```

```bash
python hello.py
# or
chmod +x hello.py && ./hello.py
```

**Useful libraries for security work:** `requests` (HTTP), `beautifulsoup4` (scraping/recon), `scapy` (packet crafting — limited without root, but useful for parsing), `pwntools` (CTF/binary exploitation, install via pip — heavier).

## Part 3 — Cybersecurity Toolkit (the practical part)

### Step 15: Build a Proper Pentest Environment with proot-distro

Termux's native packages cover a lot, but some security tools expect a full Debian/Kali-style filesystem. `proot-distro` gives you that without root:

```bash
pkg install proot-distro
proot-distro list                 # see available distros (Ubuntu, Debian, Fedora, Arch, Alpine)
proot-distro install ubuntu
proot-distro login ubuntu
# now inside Ubuntu — apt install <tool> works normally
exit
```

This is where to install heavier tools (e.g. full Metasploit, Burp-alternatives, larger wordlist-based tools) that don't play well as native Termux packages.

### Step 16: Reconnaissance

```bash
pkg install nmap whois

nmap -sV 192.168.1.1          # service/version scan
nmap -A target.com              # aggressive scan (OS, version, scripts, traceroute)
nmap -p 80,443,3306 target.com   # specific ports
nmap 192.168.1.0/24               # scan a subnet

whois domain.com                   # registration info
```

> [!note] OSINT tools `theHarvester` (email/subdomain gathering) and `sherlock` (username search across platforms) both run fine inside a `proot-distro` Ubuntu/Debian environment via `pip install`.

### Step 17: Web Application Testing

```bash
pkg install python
pip install sqlmap          # SQL injection testing
pkg install nikto              # web server vuln scanner (via proot-distro if unavailable natively)
```

```bash
sqlmap -u "http://target.com/page?id=1" --batch     # automated SQLi testing
nikto -h http://target.com                            # web server scan
```

**Directory/content discovery:**

```bash
pkg install gobuster
gobuster dir -u http://target.com -w wordlist.txt
```

**Burp Suite alternative (Burp's GUI doesn't run well in Termux):**

```bash
pip install mitmproxy
mitmproxy        # intercepting proxy — point your browser at it
```

This directly complements the PortSwigger Web Security Academy work you're already doing — you can replicate a lot of those labs' traffic inspection with `mitmproxy` or `curl -v`.

### Step 18: Password Attacks

```bash
pkg install hydra john
```

```bash
hydra -l username -P wordlist.txt ssh://target        # SSH brute force — your own box/lab only
john --wordlist=rockyou.txt hashes.txt                   # offline hash cracking
```

> [!note] Hashcat / GPU cracking `hashcat` technically installs in Termux but Android GPUs aren't exposed for cracking, so it runs CPU-only and is slow. For serious hash-cracking practice, do it on your Arch laptop's RTX 3050 instead — Termux is better for the SSH/web side of things.

### Step 19: Forensics & File Analysis

```bash
pkg install exiftool binwalk steghide file
```

```bash
exiftool image.jpg          # metadata (great for OSINT/CTF "find the location" challenges)
binwalk firmware.bin           # find embedded files/filesystems
steghide extract -sf image.jpg   # extract hidden data (steganography)
file unknown_file                  # identify file type
strings binary                       # pull readable text out of a binary (you already used this in Bandit!)
```

### Step 20: Wireless — Set Expectations

Termux **cannot** do monitor-mode WiFi attacks (aircrack-ng style) without root and a compatible external adapter — Android's WiFi stack blocks it. If wireless security interests you, that's a job for a dedicated Kali laptop/USB adapter, not your phone. Don't waste time chasing this in Termux.

### Step 21: Connecting to Practice Platforms

|Platform|How to connect from Termux|
|---|---|
|**OverTheWire**|`ssh banditN@bandit.labs.overthewire.org -p 2220` — works natively, no setup|
|**TryHackMe**|Download your OpenVPN config, then `pkg install openvpn && openvpn --config yourfile.ovpn`|
|**HackTheBox**|Same OpenVPN approach as TryHackMe|
|**PortSwigger Academy**|Browser-based — Termux isn't needed, but you can use `mitmproxy`/`curl` to inspect requests alongside it|

## Part 4 — Advanced / Automation

### Step 22: Useful Automation Scripts

**Recon-on-demand script:**

```bash
nano recon.sh
```

```bash
#!/data/data/com.termux/files/usr/bin/bash
TARGET=$1
echo "== Whois ==" && whois $TARGET
echo "== Nmap quick scan ==" && nmap -sV -T4 $TARGET
echo "== DNS ==" && dig $TARGET
```

```bash
chmod +x recon.sh
./recon.sh target.com
```

**Auto-backup of notes/scripts:**

```bash
#!/data/data/com.termux/files/usr/bin/bash
BACKUP_DIR=~/backups
DATE=$(date +%Y%m%d_%H%M%S)
mkdir -p $BACKUP_DIR
tar -czf $BACKUP_DIR/home_$DATE.tar.gz ~/ --exclude=$BACKUP_DIR
ls -t $BACKUP_DIR | tail -n +6 | xargs -I{} rm -f $BACKUP_DIR/{}
```

**Auto git push for CTF write-ups:**

```bash
#!/data/data/com.termux/files/usr/bin/bash
cd ~/ctf-writeups
git add .
git commit -m "Auto commit $(date '+%Y-%m-%d %H:%M:%S')"
git push origin main
```

### Step 23: Local Target for Practice (instead of generic DB/web servers)

Rather than setting up Apache/Postgres/MySQL just to have them running, it's more useful to host an **intentionally vulnerable app** to practice against:

```bash
pkg install python
python -m http.server 8000     # quick static file server, when you need one

# For a real practice target, use proot-distro + Docker-style vulnerable apps
# (e.g. OWASP Juice Shop, DVWA) — install Node/PHP inside your Ubuntu proot
```

`sqlite3` is worth keeping natively since it's lightweight and directly relevant to SQLi practice:

```bash
pkg install sqlite
sqlite3 test.db
```

## Troubleshooting

|Issue|Fix|
|---|---|
|"Permission denied" running a script|`chmod +x script.sh`|
|"Package not found"|`pkg update` then `pkg search name`|
|"Command not found"|`pkg search` the command name — a package likely provides it|
|Storage not accessible|`termux-setup-storage` and accept the Android prompt|
|Can't access certain dirs|`pkg install proot && termux-chroot`|

## Best Practices

**Security**

1. `pkg update && pkg upgrade` weekly
2. Use SSH keys instead of passwords once you're comfortable with `ssh-keygen`
3. Never run scripts you haven't read, even your own old ones
4. Keep your CTF/recon notes and any scope details out of public git repos

**Organization**

1. `~/projects`, `~/ctf-writeups`, `~/scripts` — separate folders, not one messy home dir
2. Git everything
3. Comment your scripts — future-you will not remember why
