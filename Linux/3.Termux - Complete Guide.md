# Termux - Complete Guide from Beginner to Advanced

A comprehensive guide to using Termux on Android - from basics to powerful advanced usage.

---

## What is Termux?

**Termux** is a powerful terminal emulator and Linux environment for Android that runs without root access.

**Purpose:**
- Turn your Android device into a portable Linux machine
- Run command-line programs and scripts
- Set up development environment on mobile
- Learn Linux/programming on the go
- Automate tasks on your phone

**Key Features:**
- No root required
- Package manager (apt/pkg)
- Access to thousands of Linux packages
- SSH client and server
- Python, Node.js, Ruby, Go support
- File system access

### Step 3: Update and Upgrade Packages

**What:** Update package repositories and installed packages

**Purpose:** Get latest software versions and security patches

**Commands:**
```bash
# Update package lists
pkg update

# Or use apt (same thing)
apt update

# Upgrade all installed packages
pkg upgrade

# Combined (recommended to run regularly)
pkg update && pkg upgrade -y
```

**Explanation:**
- `pkg update` - Downloads latest package information
- `pkg upgrade` - Installs newer versions of packages
- `-y` - Automatically answers "yes" to prompts

**When to run:** Every week or before installing new packages

### Step 5: Set Up Storage Access

**What:** Allow Termux to access phone storage

**Purpose:** Read/write files in Downloads, DCIM, etc.

**Command:**
```bash
termux-setup-storage
```

**What happens:**
- Android permission dialog appears
- Click "Allow"
- Creates `~/storage` directory with shortcuts

**After setup:**
```bash
# Access Downloads
cd ~/storage/downloads

# Access internal storage
cd ~/storage/shared

# Access camera photos
cd ~/storage/dcim
```

**Why important:** Without this, Termux can't access your phone files

## Intermediate Usage

### Step 10: Setting Up SSH

**What:** Connect to remote servers or allow connections to Termux

**Purpose:** Remote access, file transfer, server management

#### SSH Client (Connect to other servers):

```bash
# Install OpenSSH
pkg install openssh

# Connect to remote server
ssh username@server_address

# Example
ssh user@192.168.1.100

# With specific port
ssh -p 2222 user@server.com

# Copy files to server (SCP)
scp file.txt user@server:~/destination/

# Copy from server
scp user@server:~/file.txt ./
```

#### SSH Server (Connect TO Termux):

```bash
# Install openssh
pkg install openssh

# Start SSH daemon
sshd

# Check SSH is running
ps aux | grep sshd

# Find your port (usually 8022)
logcat -s 'sshd:*'

# Set password (required for SSH)
passwd

# Find your IP address
ifconfig

# From another device, connect:
# ssh -p 8022 your_termux_ip
```

**Stop SSH server:**
```bash
pkill sshd
```

### Step 22: Automation Scripts

**What:** Create useful automation scripts

**Example 1: System Backup:**
```bash
nano backup_system.sh
```

```bash
#!/data/data/com.termux/files/usr/bin/bash

# Backup script
BACKUP_DIR=~/backups
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# Backup important directories
tar -czf $BACKUP_DIR/home_$DATE.tar.gz ~/ --exclude=$BACKUP_DIR

# Keep only last 5 backups
cd $BACKUP_DIR
ls -t | tail -n +6 | xargs rm -f

echo "Backup completed: $BACKUP_DIR/home_$DATE.tar.gz"
```

**Example 2: Website Monitor:**
```bash
nano monitor_site.sh
```

```bash
#!/data/data/com.termux/files/usr/bin/bash

SITE="https://example.com"
STATUS=$(curl -o /dev/null -s -w "%{http_code}" $SITE)

if [ $STATUS -ne 200 ]; then
    termux-notification \
        --title "Site Down!" \
        --content "$SITE returned $STATUS"
fi
```

**Example 3: Auto Git Push:**
```bash

```


## Best Practices

### Security

1. **Keep updated:** Run `pkg update && pkg upgrade` weekly
2. **Use strong passwords:** For SSH, databases
3. **Backup regularly:** Important scripts and data
4. **Be careful with scripts:** Review before running
5. **Use SSH keys:** Instead of passwords when possible

### Performance

1. **Close unused processes:** Use `ps` and `kill`
2. **Clean package cache:** `pkg clean`
3. **Monitor resources:** Use `htop`
4. **Limit background tasks:** Don't run too many cron jobs

### Organization

1. **Create project folders:** `~/projects`, `~/scripts`
2. **Use version control:** Git for all code
3. **Document scripts:** Add comments
4. **Name files clearly:** Use descriptive names
